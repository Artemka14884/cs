# TACTICAL — прототип мультиплеерного FPS-шутера

Прототип шутера от первого лица в стиле CS2: Node.js + Express + WebSocket (`ws`) на
сервере, Three.js на клиенте. Один сервер держит до 10 игроков, две карты,
3 вида оружия, осколочные и дымовые гранаты, лобби-чат и общий чат матча.

## Структура проекта

```
cs2-clone/
├── server.js          # сервер: Express + WebSocket, вся игровая логика
├── package.json
├── render.yaml         # конфиг для автодеплоя на Render
├── public/
│   └── index.html      # весь клиент (меню, чат, Three.js, HUD, управление)
└── README.md
```

## Запуск локально

Требуется Node.js версии 18 или новее.

```bash
cd cs2-clone
npm install
npm start
```

Сервер поднимется на `http://localhost:3000` (или на порту из переменной `PORT`,
если она задана). Откройте этот адрес в браузере — можно в нескольких вкладках
или с других устройств в одной сети (тогда вместо `localhost` укажите IP машины,
на которой запущен сервер).

## Деплой на Render.com

### Вариант 1 — через render.yaml (Blueprint), автоматически

1. Залейте папку проекта в репозиторий на GitHub/GitLab.
2. В Render зайдите в **New → Blueprint**, укажите репозиторий.
3. Render найдёт `render.yaml` и сам создаст веб-сервис со следующими параметрами:
   - `buildCommand: npm install`
   - `startCommand: npm start`
   - план `free` (можно поменять на платный для стабильного аптайма)
4. Нажмите **Apply** — через пару минут сервис будет развёрнут.

### Вариант 2 — вручную, без Blueprint

1. **New → Web Service**, подключите репозиторий с проектом.
2. Настройки:
   - **Environment**: `Node`
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Instance Type**: Free (или любой другой)
3. Никаких дополнительных переменных окружения задавать не нужно — сервер сам
   читает порт из `process.env.PORT`, который Render передаёт автоматически.
4. После деплоя Render выдаст публичный URL вида
   `https://your-service.onrender.com` — просто откройте его в браузере.

### Важно про бесплатный план Render

- На бесплатном плане сервис "засыпает" после ~15 минут неактивности и
  "просыпается" ~30–60 секунд при первом запросе — это нормально для прототипа,
  но не подходит для соревновательной игры "в проде". Для игры с друзьями без
  задержки на пробуждение сервера используйте платный план (`starter` и выше).
- WebSocket-соединения на Render работают "из коробки", отдельная настройка
  не требуется.
- Пропускная способность и число одновременных подключений на free-плане
  ограничены — для 8–10 игроков этого достаточно.

## Как играть

1. Открыть сайт → ввести никнейм → **Играть**.
2. Управление:
   - `WASD` — движение, `Space` — прыжок, мышь — обзор.
   - `1` / `2` / `3` — автомат / пистолет / нож.
   - `4` / `5` — осколочная / дымовая граната (бросок — ЛКМ).
   - ЛКМ (удерживать для автомата) — стрельба.
   - `R` — перезарядка.
   - `Enter` — открыть чат матча, `Esc` — освободить курсор мыши.
3. Раунд длится 5 минут (Deathmatch), после чего карта автоматически меняется
   на следующую в ротации (Dust2-like → Mirage-like → Dust2-like → ...).
4. При разрыве соединения клиент сам переподключается к серверу с
   постепенно увеличивающейся паузой (до 8 секунд) и автоматически
   возвращает игрока в матч под тем же никнеймом.

## Известные упрощения прототипа (что стоит доработать дальше)

- Позиция игрока валидируется сервером только по границам карты — полноценной
  server-side проверки на "прохождение сквозь стены" (anti-cheat) нет.
  Для продакшена стоит перенести коллизии движения на сервер.
- Хитскан-стрельба и физика гранат — полностью серверные и авторитетные
  (сервер решает, было попадание или нет), а рендер — чисто визуальный.
- Дымовая граната сейчас не блокирует видимость/рейкаст — только визуальный
  эффект. Для полноценного геймплея можно добавить проверку "прямая
  видимость проходит через дым → снизить точность/скрыть silhouette".
- Нет системы команд (Terrorists/CT) и покупки оружия — реализован режим
  свободного Deathmatch со всеми видами оружия сразу.
- Нет звука. Добавление позиционного аудио (Web Audio API) — хорошее
  следующее улучшение.
